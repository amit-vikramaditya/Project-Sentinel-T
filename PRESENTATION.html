The Problem

CAN protocol (ISO 11898, 1986) — no authentication, no encryption, no sender ID.

🔓 Broadcast by Design

Every node sees every message. Messages identified by CAN ID, not sender address. Any node can impersonate any other node.

📡 Expanded Attack Surface

OBD-II ports, TCUs with cellular, infotainment (WiFi/BT/USB), V2X modules, aftermarket dongles — all pathways to the CAN bus.

Reference: Miller & Valasek (2015) — remote compromise of Jeep Cherokee steering, braking, transmission via cellular → TCU → CAN bus chain.

Why Cryptographic MACs Fail on CAN

📦 8-Byte Payload Limit

Truncated HMAC consumes ≥ 50% of data payload. Unacceptable bandwidth trade-off for safety-critical signals.

🔧 Legacy Hardware

Hundreds of millions of vehicles use 8/16-bit MCUs (Renesas RL78, NXP S12) without crypto accelerators. No firmware path.

⏱️ Real-Time Deadline

CAN cycles: 1–10 ms. Crypto latency on constrained MCUs risks violating hard real-time safety requirements.

🔑 Key Management

70–100 ECUs, multiple Tier-1 suppliers, full vehicle lifetime. Key distribution and rotation is a supply-chain nightmare.

→ We need physical-layer authentication, not data-layer encryption.

The Core Insight

Real ECU — Crystal Oscillator

  • Quartz crystal has temperature-dependent frequency
  • Clock drift is correlated over time
  • Unique per device (manufacturing tolerance)
  • Physically bounded by thermal mass

Modeled as: sinusoidal thermal drift + Ornstein-Uhlenbeck mean-reverting jitter

VS

Software Attacker — OS Timer

  • Relies on sleep() / usleep()
  • Jitter is uncorrelated (memoryless)
  • Gaussian noise from scheduling
  • Cannot replicate physical silicon behavior

This structural difference is physically unforgeable in software.

The Kernel Tap: Why time.time() Fails

❌ time.time()

Timestamp recorded after user-space scheduling. OS jitter: 50–500 µs. Same magnitude as the drift signal. Useless.

✅ SO_TIMESTAMP

Timestamp recorded inside the kernel at socket buffer delivery. Before scheduling. Microsecond resolution. Critical enabler.

# can_receiver.py — The Kernel Tap
SO_TIMESTAMP = 29
self.sock.setsockopt(SOL_SOCKET, SO_TIMESTAMP, 1)

msg, ancdata, flags, addr = self.sock.recvmsg(16, cmsg_capacity)

for cmsg_level, cmsg_type, cmsg_data in ancdata:
    if cmsg_level == SOL_SOCKET and cmsg_type == SCM_TIMESTAMP:
        seconds, microseconds = struct.unpack("qq", cmsg_data)
        kernel_timestamp = seconds + (microseconds / 1e6)

Phase-Velocity Kalman Filter

State Vector:   xk = [ φk , φ̇k ]T
φk = phase offset (seconds)  |  φ̇k = frequency drift (s/interval)

State Transition (F)

F = │ 1  1 │
    │ 0  1 │

"Drift rate changes slowly
 between 10ms measurements"

Recursion (per packet)

1. PREDICT
   x_pred = F @ x
   P_pred = F @ P @ Fᵀ + Q

2. MEASURE
   residual = z - H @ x_pred

3. UPDATE
   K = P_pred @ Hᵀ @ S⁻¹
   x = x_pred + K × residual

Observation: H = [1, 0] — only phase offset is measured. Key output: the residual measures how surprised the filter is by each packet.

Filter Tuning: Q & R

1×10⁻¹²
Q (Process Noise)

"Trust the model strongly. Physical drift is extremely slow."

1×10⁻¹⁰
R (Measurement Noise)

"Individual measurements are noisy (~10µs σ). Don't overreact."

10⁻²
Q / R Ratio

"Heavy damping. Sees through OS jitter to the physical drift."

High R/Q ratio → slow convergence, strong noise rejection → filter recovers the thermal drift signal under Linux scheduling noise.

if update_count < 10:

Conclusions

  • Kernel timestamping is essential. SO_TIMESTAMP reduces noise by 1–2 orders of magnitude vs time.time().
  • The Kalman Filter works. 2-state PV filter (Q=1×10⁻¹², R=1×10⁻¹⁰) separates correlated drift from uncorrelated jitter.
  • Clear detection boundary. 200 µs threshold cleanly separates 0.53 µs (ECU) from 350–800 µs (attacker).
  • Real-time capable. < 0.04 ms latency leaves > 99.6% of the 10 ms CAN cycle untouched.
  • Cloud validation is sufficient for Phase 1. Azure + vcan0 proved algorithmic viability.

1.88× SNR confirms: physical clock drift is distinguishable from software timing, even through Linux kernel jitter.
